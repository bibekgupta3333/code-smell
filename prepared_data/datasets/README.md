## Label sources

- **annotated/** — labels are extracted by scanning every `SmellyAnnotated/*`
  file for inline comments mentioning a canonical Fowler smell (e.g. `// Long
  Method`, `# Primitive Obsession`). Each label carries an exact line range:
  - `inline` — comment trails a code line → `line_start == line_end`.
  - `block` — comment-only line introducing a multi-line block (e.g. a group
    of fields) → expanded to the next blank/comment line.
  - `method-scope` — comment introduces a method header → expanded to the
    matching `}` (Java/JS/C++) or to the dedent (Python).
- **unannotated/** — `SmellyUnannotated/*` source has all smell comments
  stripped, so it carries no inline labels. Each record is paired with a
  `ground_truth` array sourced from
  `data/datasets/SmellyCodeDataset/Analysis/GroundTruthLevel/Cleaned_GroundTruth.csv`
  (478 method-level entries from the dataset authors). Use this for grading
  blind LLM output.

```jsonc

  "language":   "java" | "python" | "javascript" | "cpp",
  "class_name": "Cashier",
  "file_path":  "Java/SmellyAnnotated/Cashier.java",
  "loc":        175,
  "source_code": "<full file contents>",
  "annotations": [                       // populated for annotated/, [] for unannotated/
    {
      "smell_type": "Primitive Obsession",
      "category":   "Bloaters",
      "method":     "Entire Class",
      "line_start": 12,
      "line_end":   13,
      "evidence":   "    // Primitive Obsession",
      "annotation_kind": "block"         // inline | block | method-scope
    }
  ],
  "ground_truth": [                      // populated for unannotated/ only
    {"smell_type": "Large Class", "category": "Bloaters",
     "method": "Entire Class", "description": "..."}
  ],
  "metadata": { "variant": "annotated" | "unannotated", "num_annotations": 23, "num_lines": 175 }
}
```

## Smell taxonomy

23 canonical smells across Fowler's 5 categories — see
[summary.json](summary.json) `smell_taxonomy` for the full list and the alias
table that normalises spelling variants found in source comments
(e.g. `Message Chain` → `Message Chains`, `Middleman` → `Middle Man`).

## How to use

### Train / evaluate an LLM

1. **Inputs** → `unannotated/{train,val,test}.json` (`source_code` field).
2. **Targets** → matched record in `annotated/{train,val,test}.json`
   (`annotations` field) for line-level scoring, or `ground_truth` from the
   unannotated record for method-level scoring.
3. Records in both variants share the same `sample_id` (modulo path prefix),
   so they can be joined directly.

### Smell coverage at a glance

See [summary.json](summary.json) → `annotated.per_language_smell_counts` and
`unannotated.per_language_smell_counts`.

### Per-occurrence CSVs (for P/R/F1)

`<variant>/occurrences/<lang>.csv` — one row per smell instance.
Columns: `language, file_path, class_name, smell_type, category, method, line_start, line_end, annotation_kind, evidence`.

For **annotated**: `line_start`/`line_end` are populated → use for line-level
or method-scope IoU matching against LLM predictions.
For **unannotated**: line columns are empty (CSV ground truth is method-level
only) → match by `(file_path, class_name, method, smell_type)`.
